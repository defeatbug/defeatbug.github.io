# Agent 是什么？

**阅读时长：** 8 分钟 | **类型：** 概念

## 1. 从"聊天"到"做事"

普通的 LLM 调用：

```
你: "今天北京天气怎么样？"
LLM: "抱歉，我没有实时数据。"
```

加了 Agent 之后：

```
你: "今天北京天气怎么样？"
Agent: （自动调用天气查询工具 → 拿到数据 → 回答）"北京今天晴，22度。"
```

**Agent = 模型 + 工具 + 推理循环。**

## 2. Agent 的决策循环

```
      ┌──────────────────────┐
      │  用户提问             │
      └─────────┬────────────┘
                ↓
      ┌──────────────────────┐
      │  LLM 思考             │ ← "我需要什么工具？"
      └─────────┬────────────┘
                ↓
         ┌──────┴──────┐
         ↓              ↓
    需要工具         不需要工具
         ↓              ↓
   ┌──────────┐   ┌──────────┐
   │ 调用工具  │   │ 直接回答  │
   └────┬─────┘   └──────────┘
        ↓
   ┌──────────┐
   │ 拿到结果  │
   └────┬─────┘
        ↓
   ┌──────────┐
   │ LLM 再思考│ ← "结果够了吗？"
   └────┬─────┘
        ↓
   ┌────┴────┐
   ↓         ↓
 够了      不够 → 再调工具
   ↓
 最终回答
```

这就是 Agent 的核心。它自己决定**什么时候调工具、调哪个工具、调完了怎么用结果**。

## 3. 工具怎么给 Agent？

在 LangChain 里：

```python
from langchain.agents import create_agent

agent = create_agent(
    model=model,        # 大脑
    tools=tools,        # 工具箱
    system_prompt="...", # 行为规范
)
```

关键是 **tools 列表**。Agent 会把每个工具的名字、描述、参数发给 LLM，让 LLM 决定什么时候用哪个。

## 4. LangChain Agent 的演变

| 时期 | 方式 | 状态 |
|------|------|------|
| 早期 | `initialize_agent()` + AgentType | 已废弃 |
| 现在 | `create_agent()` + LangGraph 底层 | 推荐 |
| 高级 | `StateGraph` 手动编排 | 复杂场景 |

现在的 `create_agent()` 封装了最佳实践，大多数情况够用。

## 5. Agent 不做什么

- Agent **不是在"思考"** — 它只是在预测下一个 token（这点很重要）
- Agent **不会真的"理解"你的目标** — 它被 system_prompt 和 tools 约束
- Agent **可能犯错** — 调了错的工具、陷入循环、或者给不出答案

良好的 system_prompt 和工具设计能大幅降低这些问题。
